Linear search sorts checks through an array from left to right, so when you look for a number, e.g. 67, from a list of primes it will have to run a check on every index from 0-18 wher it will find 67.

Therefore, a binary search is more efficient, where you start by checking if the item at the middle index matches the search. If not is it bigger or smaller? Then divide the remaining half by 2 again.
